c++头文件 head file

函数声明和定义，函数的声明一般放在headfile中，然后在用的时候include头文件，函数只能定义一次，一般放在cpp文件中。

预处理# 符号

两种头文件保护方式

  • #pragma once
  • #ifndef #define #endif

#include 引号和尖括号

方括号只用于编译器的include路径，引号用于所有，引号会从当前相对位置开始寻找，在从include路径寻找

指针 pointer

指针就是一个内存地址，一个包含地址的变量、整形变量。

引用

对某个已存在的变量的引用，与指针不同，必须引用一个已经存在的变量

class 和 struct 的区别

class主要作为对象的实现

struct主要用于数据结构的实现

全局static 只属于这个cpp文件，其它cpp不可访问

class static 属于这个类

静态局部变量的用法 local static

c++虚函数

1、多态（polymorphism），即基类指针或者引用，可能指向基类对象，也可能指向派生类对象，会根据运行时具体指向的对象，来调用不同的函数。

2、如果没有多态，则什么类型的指针或者引用，就会调用什么类型的对象的函数。

3、每个包含虚函数的类（也叫虚类），都有一个虚函数表（简称虚表），表项是一个虚函数的地址。

4、如果基类是虚类，则继承类也是虚类。

5、虚类的对象最前面都是一个虚表指针，指向类的虚表。

虚函数表包含基类中所有虚函数的映射

对象的虚表指针用来指向自己所属类的虚表，虚表中的指针会指向其继承的最近的一个类的虚函数

首先 需要额外的内存才能将信息存储到虚函数表中，该函数包含实际基类中指向虚函数表的成员指针

其次，每当我们调用虚函数时，需要遍历虚函数表以查看该映射哪个函数。

c++接口

pure virtual function（纯虚函数） virtual return function() =0 相当于接口

必须由子类来实现 接口。

这样做的好处是 所有的子类都会实现该方法。 保证了多态。

接口和抽象类的区别

https://jingyan.baidu.com/article/ceb9fb10703f7f8cac2ba05e.html

抽象类要被子类继承，接口要被类实现。

接口只能做方法声明，抽象类中可以作方法声明，也可以做方法实现。

接口里定义的变量只能是公共的静态的常量，抽象类中的变量是普通变量。

接口是设计的结果，抽象类是重构的结果。

抽象类和接口都是用来抽象具体对象的，但是接口的抽象级别最高。

抽象类可以有具体的方法和属性，接口只能有抽象方法和不可变常量。

抽象类主要用来抽象类别，接口主要用来抽象功能。

指针和引用的区别

https://blog.csdn.net/l477918269/article/details/90233908

引用是别名、指针是地址

指针可以重新赋值，指向另一个地址，但是引用就是别名，与某个对象绑定之后，不能改变。

程序需要为指针变量分配内存，不需要为引用分配内存、因为它就是一个别名，引用不能为空。

C++11中引入了三种智能指针，分别是shared_ptr、weak_ptr和unique_ptr
